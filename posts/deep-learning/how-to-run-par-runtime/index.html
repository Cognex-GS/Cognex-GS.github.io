    Written by Alex Choi, Oct. 14, 2020.

    Pre-requisites

    • Visual Studio 2019

    1. ParRuntimeTest.exe 실행 파일

    • Prebuilt:

      • NAS: http://10.99.160.30:5000/ML Data/015_COI_rnd_tf/_research_result/ParRuntimeTest_20201014_v1.zip
      • OneDrive: Link

    • Source code: https://github.com/sualab/Par / branch: [Branch_SearchMode]

    • Argument:

      • evalType: [seg/cls] 0: classification, 1: segmentation.
      • gpuIndex: [seg/cls] 사용할 gpu index.
      • inputWidth/inputHeight: [seg/cls] seg에서는 runtime 시의 patch size를 의미하고, cls에서는 input image patch size를 의미함.
      • patchSize: [seg] seg에서 추출할 patch size. Seg 결과를 cls에서 활용할 경우 cls에서 사용할 patch size가 지정되어야 함.
      • inputChannel: [seg/cls] model과 input image channel. Side: 6, Front: 3.
      • targetNumberOfPatchMin / targetNumberOfPatchMax: [seg] search mode에서 사용하는 샘플 단위 패치 총 수의 평균의 최소값/최대값.
      • segThreshold / claThreshold: [seg/cls] model에 나온 확률 값을 NG/OK로 구분하기 위한 threshold. [0,1] 사이의 범위 지정. 0 미만의 값으로 지정 시 search mode로 동작함. 이때 아래의 probThresholdBegin과 probThresholdEnd 값이 지정되어 있어야 함. 두 값을 지정하지 않으면 두 값은 default 값으로 동작함.
      • probThresholdBegin / probThresholdEnd: [seg/cls] search mode에서 사용하는 threshold의 search 범위의 최소값/최대값.
      • pairSeparatorFirst / pairSeparatorSecond: [seg/cls] diffuser on/off 등 이미지 짝을 맞추기 위한 값. side에서는 ["d-off” / “d-on"]로 front에서는 [“_Front” / “”]로 설정하나 최신 파일 네이밍 규칙에서는 바뀔 수 있음.
      • targetYieldMin / targetYieldMax: [cls] search mode에서 threshold를 찾기 위해 ok-only-wild dataset을 활용하여 계산하는 수율의 최소값/최대값.
      • inputFolder / maskFolder: [seg/cls] search mode가 아닌 일반 모드에서 필요로 하는 image/mask 폴더의 경로.
      • okInputImagePath / ngInputImagePath: [cls] 이 두 값이 모두 지정되어 있으면 okInputImagePath의 경로를 이용하여 모델의 수율을 계산하고, ngInputImagePath의 경로를 이용하여 모델의 미검률을 계산함.
      • segModelPath / claModelPath: [seg/cls] model path.

    • Result: 기본 경로는 ParRuntimeTest.exe 파일의 상위 폴더에 outs 라는 폴더에 실행한 시간으로 결과가 출력됨.

    2. Segmentation model 실행 방법

    • Model: Link or 연구팀에게 요청 or Woody님에게 요청

    • Dataset:

      • OK dataset
        • 목적: 샘플 단위 수율이 목표치 이내에 들도록 segmentation threshold 설정.
        • 경로: NAS[10.99.160.32:5000]/Archive-Research/999_project/015_COI_rnd_tf/a102du-"black/red/white"/whole/ok-only-wild 또는 연구팀에 요청.
      • NG dataset
        • 목적: 위의 OK dataset을 활용해 얻은 threshold로 NG dataset을 처리하고, 이때 패치 단위 혹은 샘플 단위의 NG2OK(미검) 숫자를 고려하여 미검이 가장 적은 모델을 선정.
        • 경로: NAS[10.99.160.32:5000]/Archive-Research/999_project/015_COI_rnd_tf/a102du-"black/red/white"/whole/ng-biased-wild 또는 연구팀에 요청.

    • Argument settings 

      --evalType 1
--targetNumberOfPatchMin 50
--targetNumberOfPatchMax 80
--segThreshold 0.534375
--probThresholdBegin 0.3
--probThresholdEnd 0.7
--pairSeparatorFirst "d-off"
--pairSeparatorSecond "d-on"
--inputFolder "E:\999_project\015_COI_rnd_tf\a102du-black\whole\ok-only-wild\image/"
--maskFolder "E:\999_project\015_COI_rnd_tf\a102du-black\whole\ok-only-wild\mask\non-ok.2class/"
--segModelPath "E:\999_project\015_COI_rnd_tf\_research_result\a102du_side_seg_models\20201013/102du_black_side_seg_epoch92_resnet18_20201013.model"
--gpuIndex 0
--patchSize 128
--inputWidth 3520
--inputHeight 512
--inputChannel 6

    위와 같은 옵션을 갖는 경우, 아래와 같이 실행:

    --evalType 1 --targetNumberOfPatchMin 50 --targetNumberOfPatchMax 80 --segThreshold 0.534375 --probThresholdBegin 0.3 --probThresholdEnd 0.7 --pairSeparatorFirst "d-off" --pairSeparatorSecond "d-on" --inputFolder "E:\999_project\015_COI_rnd_tf\a102du-black\whole\ok-only-wild\image/" --maskFolder "E:\999_project\015_COI_rnd_tf\a102du-black\whole\ok-only-wild\mask\non-ok.2class/" --segModelPath "E:\999_project\015_COI_rnd_tf\_research_result\a102du_side_seg_models\20201013/102du_black_side_seg_epoch92_resnet18_20201013.model" --gpuIndex 0 --patchSize 128 --inputWidth 3520 --inputHeight 512 --inputChannel 6
    • Results
      • 결과 패치가 ParRuntimeTest.exe/../outs/시간/ 에 저장됨.
      • 이 결과는 아래의 classification model을 평가하는데 반드시 필요.

    3. Classification model 평가 방법

    • Model: Link or 연구팀에게 요청 or Woody님에게 요청.

    • Dataset:

      • OK dataset: 위의 Segmentation model 실행 방법을 참고하여 ok-only-wild 데이터셋을 처리하고 나온 결과물인 이미지의 폴더를 지정.
      • NG dataset: 위의 Segmentation model 실행 방법을 참고하여 ng-biased-wild 데이터셋을 처리하고 나온 결과물인 이미지의 폴더를 지정.

    • Argument settings 

      --evalType 0
--targetYieldMin 0.82
--targetYieldMax 0.85
--probThresholdBegin 0.2
--probThresholdEnd 0.8
--inputFolder ""
--okInputImagePath "E:\999_project\015_COI_rnd_tf\seg patch for cls\side\black\20201013\20201013_233934_a102du_side_black_patch_ok_only/"
--ngInputImagePath "E:\999_project\015_COI_rnd_tf\seg patch for cls\side\black\20201013\20201013_232512_a102du_side_black_patch_ng_biased/"
--pairSeparatorFirst "d-off"
--pairSeparatorSecond "d-on"
--claModelPath "E:\999_project\015_COI_rnd_tf\_research_result\20201013_side_cls_models\black_model/"
--claThreshold -1
--gpuIndex 0
--inputChannel 6

    위와 같은 옵션을 갖는 경우, 아래와 같이 실행:

    --evalType 0 --targetYieldMin 0.82 --targetYieldMax 0.85 --probThresholdBegin 0.2 --probThresholdEnd 0.8 --inputFolder "" --okInputImagePath "E:\999_project\015_COI_rnd_tf\seg patch for cls\side\black\20201013\20201013_233934_a102du_side_black_patch_ok_only/" --ngInputImagePath "E:\999_project\015_COI_rnd_tf\seg patch for cls\side\black\20201013\20201013_232512_a102du_side_black_patch_ng_biased/" --pairSeparatorFirst "d-off" --pairSeparatorSecond "d-on" --claModelPath "E:\999_project\015_COI_rnd_tf\_research_result\20201013_side_cls_models\black_model/" --claThreshold -1 --gpuIndex 0 --inputChannel 6
    • Results: outs 폴더에 cls_result.txt 파일에 기록됨.
